# RUST project patch rules

You are patching an existing React + TypeScript + Vite + PWA app.

Do not redesign loosely.
Do not stop at description.
Edit the actual code.

Critical rules
- Keep the project deployable on Vercel.
- Keep the project cloud-ready for Supabase.
- Keep Dutch UI.
- Keep mobile-first portrait behavior.
- Avoid horizontal overflow.
- Use safe-area aware spacing.
- Prefer clear, stable UI over flashy changes.
- Fix bugs first, then polish visuals.
- Do not remove working features unless they block requested fixes.
- Replace demo content that confuses users.
- Build for iPhone and Android portrait, with tablet fallback.
- When patching Vercel or Supabase related files, preserve standard conventions.
